03 · Reverse Proxy

nginx-rp — Reverse Proxy Layer

nginx-rp นั่งอยู่หน้า app servers รับทุก request ผ่าน port 8080 แล้วส่งต่อพร้อมจัดการ headers, logging และ security

Reverse Proxy — Header Manipulation Flow
01

รับ Request จาก Browser ผ่าน :8080

nginx-rp รับ connection บน port 8080 client มองเห็นแค่ nginx-rp ไม่รู้ว่า app จริงอยู่ที่ไหน

02

เพิ่ม Headers สำคัญ

เพิ่ม X-Real-IP, X-Forwarded-For, X-Forwarded-Proto เพื่อให้ backend รู้ IP จริงของ client แม้ผ่าน proxy

03

ส่งต่อไปยัง app1 ผ่าน proxy_pass

ใช้ proxy_pass http://app1:80 ส่ง request ไปยัง app container ใน docker network โดยตรง

04

Log ทุก Request

บันทึก access log แยกสำหรับ reverse proxy layer ทำให้ debug ได้ง่ายว่า request มาจากไหน ผ่านอะไรบ้าง

💡 Demo นี้: nginx-rp รับ request จาก port 8080 และส่งต่อไปยัง app1 เท่านั้น เพื่อแสดงให้เห็น reverse proxy แบบ 1:1 ชัดๆ — ต่างจาก nginx-lb ที่กระจายไปทั้ง 3 servers
04 · Load Balancer

nginx-lb — Load Balancer Layer

nginx-lb รับ traffic จาก port 80 แล้วกระจายไปยัง app1/2/3 ด้วย least_conn algorithm สามารถรองรับ server down ได้โดยอัตโนมัติ

Load Balancer — Traffic Distribution (least_conn)
✅ Failover อัตโนมัติ: ถ้า app2 down ระหว่างทำงาน nginx-lb จะหยุดส่ง request ไปยัง app2 โดยอัตโนมัติ และกระจายให้ app1 กับ app3 แทน — ไม่มี downtime สำหรับ user
05 · Request Flow

Request Journey — ตามรอย packet

ดูว่า request แต่ละแบบเดินทางอย่างไรผ่าน stack ตั้งแต่ browser จนถึง PHP และกลับมา

PATH A — via Load Balancer (port 80)

1

Browser → GET http://localhost:80/

DNS resolves → docker port mapping → nginx-lb container รับ connection

2

nginx-lb ตัดสินใจส่งไป server ไหน (least_conn)

เช็ค active connections ของ app1/app2/app3 → เลือก server ที่ว่างที่สุด → proxy_pass http://appN:80

3

appN:80 (nginx web server) รับ request

nginx ใน app container รับ request → ตรวจว่า path ลงท้าย .php → pass ไปยัง php-fpm

4

PHP-FPM execute index.php

PHP อ่าน $_ENV['SERVER_NAME'] และ $_SERVER['SERVER_ADDR'] → สร้าง HTML แสดงชื่อ server

5

Response ย้อนกลับ: PHP → nginx app → nginx-lb → Browser

HTML response ไหลกลับผ่าน stack เดิม browser เห็นหน้าเว็บของ server ที่รับ request นั้น

PATH B — via Reverse Proxy (port 8080)

1

Browser → GET http://localhost:8080/

Port 8080 mapped ไปยัง nginx-rp container โดยตรง

2

nginx-rp เพิ่ม headers และ proxy ไป app1

เพิ่ม X-Real-IP, X-Forwarded-For, X-Proxy-By: nginx-rp แล้ว proxy_pass http://app1:80

3

app1 รับ request + headers ที่ nginx-rp เพิ่มมา

PHP อ่าน $_SERVER['HTTP_X_REAL_IP'] เพื่อรู้ IP จริงของ client และแสดงใน page

4

Response กลับผ่าน nginx-rp → Browser

Browser เห็นแค่ nginx-rp เป็นต้นทาง ไม่รู้ว่า app1 อยู่ที่ไหน — นั่นคือ Reverse Proxy ทำงาน

docker-compose.yml yaml 
version: '3.8'

services:

  # ── NGINX Load Balancer (port 80) ────────────────────────────────
  nginx-lb:
    image: nginx:alpine
    container_name: nginx-lb
    ports:
      - "80:80"
    volumes:
      - ./nginx-lb/nginx.conf:/etc/nginx/conf.d/default.conf:ro
    depends_on:
      - app1
      - app2
      - app3
    networks:
      - webnet
    restart: unless-stopped

  # ── NGINX Reverse Proxy (port 8080) ──────────────────────────────
  nginx-rp:
    image: nginx:alpine
    container_name: nginx-rp

      - "8080:80"

nginx-lb/nginx.conf nginx 
# ═══════════════════════════════════════════════
# NGINX LOAD BALANCER — port 80
# กระจาย traffic ไปยัง app1, app2, app3
# ═══════════════════════════════════════════════

upstream backend {
    # Algorithm: least_conn — ส่งไป server ที่ active connections น้อยสุด
    # เปลี่ยนเป็น ip_hash; เพื่อ session sticky
    least_conn;

    server app1:80;   # nginx web server ใน app1 container
    server app2:80;   # nginx web server ใน app2 container
    server app3:80;   # nginx web server ใน app3 container

    keepalive 32;     # connection pool ลด TCP handshake overhead
}

server {
    listen       80;
    server_name  localhost;

    # ── ส่งทุก request ไปยัง upstream ──────────
    location / {
        proxy_pass         http://backend;
        proxy_http_version 1.1;
        proxy_set_header   Connection        "";
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-By    nginx-lb;
    }

    # ── Nginx status endpoint ───────────────────
    location /lb-status {
        stub_status on;
        access_log  off;
    }
}
nginx-rp/nginx.conf nginx 
# ═══════════════════════════════════════════════
# NGINX REVERSE PROXY — port 8080
# ส่ง request ไปยัง app1 พร้อมเพิ่ม headers
# ═══════════════════════════════════════════════

server {
    listen      80;
    server_name localhost;

    # Custom log format — รวม X-Real-IP ด้วย
    access_log /var/log/nginx/rp-access.log;

    location / {
        # ส่งไปยัง app1 เสมอ (reverse proxy 1:1)
        proxy_pass         http://app1:80;
        proxy_http_version 1.1;
        proxy_set_header   Connection        "";

        # ── Headers ที่ reverse proxy เพิ่มให้ ────────
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
        proxy_set_header   X-Proxy-By        nginx-rp;

        # ── Timeout settings ──────────────────────────
        proxy_connect_timeout 10s;
        proxy_read_timeout    30s;
        proxy_send_timeout    30s;

        # ── Add response header (แสดงว่าผ่าน RP) ─────
        add_header         X-Served-Via  "nginx-rp";
    }

    location /rp-status {
        stub_status on;
        access_log  off;
    }
}
app/nginx.conf nginx 
# ═══════════════════════════════════════════════
# NGINX WEB SERVER — inside app container
# รับ HTTP request และ pass .php → php-fpm
# ═══════════════════════════════════════════════

server {
    listen      80;
    server_name localhost;
    root        /var/www/html;
    index       index.php index.html;

    # ── Serve static files โดยตรง ──────────────
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    # ── PHP files → FastCGI → php-fpm ──────────
    location ~ \.php$ {
        fastcgi_pass   127.0.0.1:9000;   # php-fpm socket
        fastcgi_index  index.php;
        fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;
        include        fastcgi_params;

        # ส่ง env vars ให้ PHP ด้วย
        fastcgi_param  SERVER_NAME      $hostname;
    }

    # ── Health check ────────────────────────────
    location = /health {
        access_log off;
        return 200 '{"status":"ok"}';
        add_header Content-Type application/json;
    }
}

app/Dockerfile dockerfile 
# ═══════════════════════════════════════════════
# PHP-FPM + Nginx Web Server in ONE container
# Supervisor จัดการ 2 processes
# ═══════════════════════════════════════════════
FROM php:8.2-fpm-alpine

# ── ติดตั้ง nginx และ supervisor ──────────────
RUN apk add --no-cache nginx supervisor

# ── Nginx config (web server → php-fpm) ──────
COPY nginx.conf /etc/nginx/http.d/default.conf

# ── Supervisor config (run nginx + php-fpm) ──
COPY supervisord.conf /etc/supervisord.conf

# ── PHP application files ─────────────────────
COPY www/ /var/www/html/

# ── Fix permissions ───────────────────────────
RUN chown -R www-data:www-data /var/www/html \
 && mkdir -p /run/nginx /var/log/supervisor

EXPOSE 80

# Supervisor รัน nginx และ php-fpm พร้อมกัน
CMD ["/usr/bin/supervisord", "-c", "/etc/supervisord.conf"]
app/supervisord.conf ini 
[supervisord]
nodaemon=true
logfile=/var/log/supervisor/supervisord.log
pidfile=/var/run/supervisord.pid

# ── Nginx web server ───────────────────────────
[program:nginx]
command=nginx -g "daemon off;"
autostart=true
autorestart=true
stderr_logfile=/var/log/supervisor/nginx-err.log
stdout_logfile=/var/log/supervisor/nginx-out.log
priority=10

# ── PHP-FPM ────────────────────────────────────
[program:php-fpm]
command=php-fpm8.2 -F
autostart=true
autorestart=true
stderr_logfile=/var/log/supervisor/phpfpm-err.log
stdout_logfile=/var/log/supervisor/phpfpm-out.log
priority=5
